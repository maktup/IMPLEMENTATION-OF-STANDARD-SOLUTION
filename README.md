# CONTEXT: 
- Hybrid iPaaS = 'Hybrid Integration Platform as a Service'.
- IBM acquired WebMethods  from SOFTWARE AG, and currently: IBM WEBMETHODS HYBRID INTEGRATION (WMHI) is IBM's flagship offering as a modern HYBRID iPaaS.
- A 'HYBRID iPaaS' allows you to INTEGRATE EVERYTHING with everything without moving sensitive DATA externally if you don't want to, maintaining control, security, and governance.
- It is a UNIFIED PLATFORM that covers everything from a single CONSOLE:
  + Appliation Integration
  + APIs
  + Messaging & Events
- It is called a HYBRID INTEGRATION PLATFORM because it allows you to:
  + Process flows in the CLOUD.
  + Process flows ON-PREMISE.
  + Connect both without moving sensitive DATA.

# OBJETIVE: 
Understanding how to implement a solution in WebMethods, utilizing their various capabilities such as WebMethod Integration (Workflow), APIC, WebMethod API Gateway, and Developer Portal, in the implementation of a REST-based solution.

The steps for the solution are:

- Implementation of a REST service, implemented as a WORKFLOW in WebMethod Integration.
- Exposure of a WORKFLOW solution implemented on IBM API Connect.
- Publication of the product on Developer Portal.
- Exposure of a WORKFLOW solution implemented on IBM API Gateway.
